# AGENTS.md

## Назначение проекта

- Эта директория содержит LaTeX-исходники диплома.
- Главная точка входа: `main.tex`.
- Общие настройки вынесены в `preamble.tex`.
- Основной текст подключается через `chapters/*.tex`.
- В `main.tex` также предусмотрено условное подключение `task.pdf` и `review.pdf`,
  но сами `.pdf` трогать нельзя.

## Жёсткие правила работы

- Никогда не изменять и не перезаписывать `.pdf` файлы в проекте.
- Документ должен собираться движком `XeLaTeX` (или `LuaLaTeX`), а не `pdfLaTeX`
  (используется `fontspec` и системный шрифт Times New Roman).
- Пользователь собирает документ через TexShop (нужно выбирать `XeLaTeX`, иначе сборка упадёт на `fontspec`).
- Если нужна техническая проверка LaTeX, использовать только безопасный режим без генерации PDF:
  `xelatex -no-pdf` с выводом в `/tmp`.
- Любые структурные изменения допускаются только после явного подтверждения пользователя.
- Под структурными изменениями понимать:
  порядок `\input` в `main.tex`,
  порядок и состав глав,
  систему заголовков,
  существенные изменения `preamble.tex`.
- По умолчанию менять только содержание.
- Не менять порядок и структуру глав без нового подтверждения.
- Не трогать заголовки и общую композицию документа, если пользователь отдельно не разрешил это.

## Требования к оформлению (ГОСТ + практики)

- Оформление ориентировано на ГОСТ 7.32-2017, ГОСТ Р 2.105-2019, ГОСТ Р 7.0.100-2018.
- Бумага: А4, книжная ориентация, печать с одной стороны.
- Поля: левое 30 мм, правое 15 мм, верхнее/нижнее 20 мм (`geometry` уже настроен).
- Шрифт: Times New Roman 14 pt; межстрочный интервал 1.5; абзацный отступ 1.25 см.
- Нумерация страниц: арабскими цифрами, сквозная, номер по центру снизу без точки;
  титульный лист входит в нумерацию, но номер на нём не печатается.
- `task.pdf` и `review.pdf` (если есть) подключаются через `pdfpages`, но не должны влиять на счётчик страниц.
- Библиография: `biblatex-gost` (numeric), ссылки в тексте в квадратных скобках; заголовок
  «СПИСОК ИСПОЛЬЗОВАННЫХ ИСТОЧНИКОВ».
- Техническая практика: модульная структура (`main.tex` + `preamble.tex` + `chapters/*.tex`),
  минимизация ручного форматирования, использование `tabularray`, `caption`, `titlesec`, `titletoc`.

## Текущие договорённости с пользователем

- Тема диплома: `Вычислительная модель расчёта термодинамических свойств воды`.
- Режим правок: только с подтверждения пользователя.
- Порядок и структура глав должны сохраняться.
- Содержательные расширения допустимы, если пользователь их прямо запросил.
- Пользователь уже подтвердил текущую новую структуру глав и схему именования
  файлов в `chapters/`.

## Предметная область диплома

- Основа вычислительной модели: стандарт `IAPWS-IF97` в редакции `2012`.
- Реализовано вычислительное ядро на Rust.
- Ядро расширено численной схемой на основе метода секущих для маршрута расчёта по паре `rhoT` на всей рабочей области.
- Поддерживаемые и упоминаемые входные параметры:
  `p`, `T`, `h`, `s`, `rho`, `x`.
- Типовые выходы:
  `v`, `rho`, `h`, `s`, `u`, `cp`, `w`, номер региона IF97.

## Архитектурный контекст, который нельзя забывать

- Ключевая идея: одно вычислительное ядро и несколько каналов доставки.
- Ядро: `if97_core`.
- Общие DTO и контракты обмена: `if97_app_api`.
- Нативный desktop-клиент: `FLTK`.
- Web/Desktop UI: `Yew/WASM + Tauri`.
- Сервисный контур: `gRPC`.
- Развёртывание: `Docker + Kubernetes`.
- Автоматизация: `GitHub Actions`.

## Семантические блоки диплома

- Математическая часть с формулами.
- Реализация программного ядра.
- Реализация десктопного приложения.
- Реализация контейнера, Kubernetes-среды и CI/CD.

## Отображение смысловых блоков на текущую TeX-структуру

- `chapters/02_intro.tex`: введение.
- `chapters/04_chapter01_domain_analysis.tex`: предметная область и постановка задачи.
- `chapters/05_chapter02_math.tex`: математическая модель и численные методы.
- `chapters/06_chapter03_core.tex`: вычислительное ядро на Rust.
- `chapters/07_chapter04_ui.tex`: пользовательские приложения.
- `chapters/08_chapter05_service.tex`: сервис, Docker, Kubernetes, CI/CD.
- `chapters/09_conclusion.tex`: заключение.
- `chapters/10_appendices.tex`: приложения.
- Неподключённые старые версии глав перенесены в `chapters/archive/`.

## Соглашение по именованию TeX-файлов

- Файлы в `chapters/` должны идти в порядке документа при обычной сортировке.
- Для основных глав использовать схему:
  `NN_chapterMM_short_name.tex`
- Для служебных частей использовать короткие имена без лишних вариаций:
  `00_title.tex`, `01_abstract.tex`, `02_intro.tex`, `03_main_part_separator.tex`,
  `09_conclusion.tex`, `10_appendices.tex`.
- Если создаются новые рабочие файлы в `chapters/`, их имя должно сохранять
  правильный порядок сортировки.
- Не возвращать старые имена файлов из `chapters/archive/` в основной каталог
  без явной причины и без синхронизации `main.tex`.

## Текущее состояние TeX-каркаса

- Основной маршрут сборки уже переведён на новую структуру глав.
- `chapters/01_abstract.tex` и `chapters/02_intro.tex` заполнены базовым текстом и
  дальше должны уточняться по мере наполнения основных глав.
- `chapters/09_conclusion.tex` пока содержит заглушку.
- Остальные новые файлы глав представляют собой структурные шаблоны с названиями
  разделов и подразделов без наполнения.
- Старые содержательные версии глав не удалены, а сохранены в `chapters/archive/`
  как черновой материал для переноса содержания вручную.
- Перед наполнением новых глав полезно сверяться с архивными версиями, но
  подключать их обратно в `main.tex` не нужно.

## Источники контекста в текущей папке

- Основной найденный локальный документ: `описание диплома.md`.
- На основе `описание диплома.md` созданы локальные опорные файлы:
  `MATH.md`, `CORE.md`, `FLTK.md`, `WASM_TAURI.md`, `SERVISE.md`.
- Эти файлы считать рабочими конспектами для следующих проходов по диплому.
- Дополнительный сохранённый промпт по ГОСТ-оформлению LaTeX-шаблона:
  `GOST_LATEX_TEMPLATE_PROMPT.md`.
- Сохранённая родмапа разработки и черновики формулировок для «Задания на ВКР»:
  `ROADMAP.md`.
- `references.bib` уже содержит базовый набор источников, включая IAPWS и
  документацию по используемым технологиям.
- Если позже появятся исходные `PROJECT.md` или `THESIS_OUTLINE.md`, их нужно
  сопоставить с текущими локальными конспектами и приоритетно устранить расхождения.

## Приоритет источников по разделам

- Для главы о предметной области в первую очередь использовать
  `описание диплома.md`, затем при необходимости `MATH.md`.
- Для математической главы основными считать `MATH.md` и `описание диплома.md`.
- Для главы о ядре основными считать `CORE.md` и `описание диплома.md`.
- Для главы об интерфейсах основными считать `FLTK.md`, `WASM_TAURI.md` и
  `описание диплома.md`.
- Для главы о сервисе и инфраструктуре основными считать `SERVISE.md` и
  `описание диплома.md`.
- Для библиографии и ссылок сверяться с `references.bib`; новые источники
  добавлять туда осознанно и предпочтительно из официальной документации.

## Практические напоминания для следующих проходов

- При дальнейшей доработке не возвращать шаблонные `lipsum`-вставки и фиктивные примеры.
- Стиль исходников: по возможности каждое предложение начинать с новой строки в `.tex`
  (это упрощает поиск ошибок и просмотр diff в Git).
- Если добавляются формулы, по возможности обходиться без лишних изменений `preamble.tex`.
- Если изменение `preamble.tex` всё же требуется, сначала проверить, нельзя ли решить задачу в рамках уже подключённых пакетов.
- Для библиографии предпочитать официальные документы IAPWS и официальную документацию используемых технологий.
- Перед любыми крупными содержательными изменениями сначала перечитать `main.tex`, `preamble.tex` и текущие главы, чтобы не разрушить композицию документа.
- Если задача состоит в наполнении диплома текстом, в первую очередь заполнять
  новые файлы из основного маршрута сборки, а не архивные версии.
- После любых изменений в структуре глав, именах файлов или правилах работы
  обязательно синхронизировать `AGENTS.md`.
