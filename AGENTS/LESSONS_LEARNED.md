# Повторяющиеся ошибки и канонические исправления

## Списки

- Нельзя использовать `\item[] -` и другие локальные имитации тире.
- Нельзя начинать пункт с `\texttt{...}` или `\path{...}`.
- Канонический формат: сначала смысловое слово, потом технический маркер.
- В промежуточных пунктах ставить `;`, в последнем - `.`.

Пример:

```tex
\begin{itemize}
  \item модуль \texttt{tokio} отвечает за асинхронную обработку запросов;
  \item ресурс \texttt{Deployment} описывает развёртывание сервиса.
\end{itemize}
```

## Формулы

- Нельзя ставить `\noindent` перед выделенными абзацами.
- Нельзя делать `\noindent` перед блоками `где`.
- Расшифровка после формулы идёт без двоеточия.
- Каждая переменная пишется с новой строки.
- После каждой строки, кроме последней, ставится `;`.

Пример:

```tex
\begin{equation}
  p = f(T).
\end{equation}

где $p$ --- давление, Па;\\
$T$ --- температура, К.
```

## Заголовки

- Нельзя оставлять круглые скобки вокруг названий технологий в `\section` и `\subsection`.
- Канонический вариант - встроить технологию в русскую фразу.

Пример:

```tex
\section{Модель параллелизма и асинхронная обработка запросов средствами Tokio}
```

## Региональные названия

- В тексте использовать только русские формы:
  `Регион 1`, `Регион 2`, `Регион 2 Метастабильный Пар`.
- Не использовать английские `Region`, `Region 2 Meta` и похожие хвосты.

## Библиография

- Для `@online` и `@report` обязательно проверять `year`.
- Авторы должны быть в `author`, а не прятаться в `title`.
- Хвосты вида `/ ...` из названий надо удалять.

Пример:

```bibtex
author = {Гидаспов, В. Ю. and Северина, Н. С.},
title  = {...},
```

## Сборка

- Готовые PDF должны попадать только в `results/`.
- Временные файлы должны лежать только в `.build/`.
- Корень проекта не должен содержать случайных `pdf`, `yml` или log-артефактов.

## Основная часть

- Отдельный лист «ОСНОВНАЯ ЧАСТЬ» в этом проекте не используется.
- Если где-то остался артефакт с таким названием, его нужно удалить, а не восстанавливать.
