# Карта проекта (file-by-file)

Этот файл предназначен для будущих агентов и быстрых проходов по проекту:
что где лежит, как собирается, как устроено разбиение глав на части и какие
рисунки/диаграммы/таблицы к чему относятся.

## Коротко о проекте

ВКР посвящена разработке вычислительной модели и программного комплекса для
расчёта термодинамических свойств воды и водяного пара по стандарту IAPWS-IF97.
Архитектура описывает единое вычислительное ядро на Rust и несколько каналов
доставки результатов: настольный UI (FLTK), кроссплатформенная оболочка
(Yew/WASM/Tauri), а также gRPC-сервис с инфраструктурой (Docker/Kubernetes, CI/CD).

## Директории и точки входа

- `source/` — единственное место редактируемых LaTeX-исходников.
- `source/main.tex` — единственная точка входа итогового документа.
- `source/preamble.tex` — общая преамбула (шрифты, ГОСТ-подобные настройки, biblatex).
- `source/chapters/` — главы и части глав.
- `data/pic/` — изображения (скриншоты UI, прочие картинки).
- `results/` — только готовые артефакты (`results/diploma.pdf` и PDF-части).
- `.build/` — временные файлы, логи, промежуточные PDF и скрипты сборки.

## Политика разбиения `.tex` по строкам

Цель: каждый редактируемый `.tex` держать в пределах ~250–300 строк.
Для этого большие главы разбиты на части `*_partNN.tex`.

Конвенция:

- `source/chapters/<chapter>.tex` — тонкая обёртка главы: содержит `\\chapter{...}` и набор `\\input{chapters/<chapter>_partNN.tex}`.
- `source/chapters/<chapter>_partNN.tex` — смысловой фрагмент: начинается с `\\section{...}` или `\\subsection{...}` (или с выделенного абзаца, если это продолжение текущего раздела).
- При разбиении стараться резать по границам секций/подсекций и не начинать часть “с середины мысли”.

Важно: часть должна компилироваться как включаемый фрагмент (без `\\documentclass`, `\\begin{document}` и т.п.).

## Сборка (в двух словах)

- Полная сборка: `.build/build.sh` (XeLaTeX + Biber, промежуточные файлы в `.build/full/`, итог `results/diploma.pdf`).
- Частичные PDF: `.build/chapters/build_chapters.sh` (компилирует отдельные фрагменты и кладёт их в `results/chapters/`).

## Карта исходников (`source/`)

### `source/main.tex`

- Роль: точка входа итоговой сборки.
- Подключает: `chapters/00_title.tex`, опционально `task.pdf`/`review.pdf` (если лежат рядом с `main.tex` в `.build/full/`), затем `01_abstract`, `\\tableofcontents`, главы 1–5, заключение, библиографию и приложения.

### `source/preamble.tex`

- Роль: оформление (Times New Roman через `fontspec`, `babel` ru/en, поля A4 30/15/20/20, 1.5 интервал, `\\parindent=1.25cm`).
- Библиография: `biblatex` + `biber`, стиль `gost-numeric`, печать через `\\printbibliography` с заголовком «СПИСОК ИСПОЛЬЗОВАННЫХ ИСТОЧНИКОВ».
- Списки: `enumitem` с тире для `itemize` (важно не городить локальные “костыли”).

### `source/references.bib`

- Роль: единая база источников для `biblatex` (текущий объём: 20 записей).
- Конвенции по качеству полей см. `AGENTS/AGENTS.md` и `AGENTS/СПИСОК ЛИТЕРАТУРЫ.md`.

## Карта сборки (`.build/`)

### `.build/build.sh`

- Делает чистую сборку в `.build/full/` (копирует `source/main.tex`, `source/preamble.tex`, `source/references.bib`, каталог `source/chapters/` целиком).
- Запускает `xelatex` → `biber` → `xelatex` → `xelatex`.
- Копирует `main.pdf` в `results/diploma.pdf`.
- Запускает `.build/chapters/build_chapters.sh` для генерации `results/chapters/*.pdf`.

### `.build/chapters/build_chapters.sh`

- Компилирует части документа в отдельные PDF и складывает в `results/chapters/`.
- Важный нюанс: создаёт симлинк `.build/chapters/chapters -> source/chapters`, чтобы работали `\\input{chapters/...}` из обёрток и частей.
- Список компилируемых фрагментов задан явно (при добавлении новых `*_part*.tex` его нужно обновлять).

### `.build/chapters/chapter_wrapper.tex`

- Мини-шаблон для сборки “фрагмента” в отдельный PDF: задаёт преамбулу и делает `\\input{\\ChapterFile}`, затем печатает библиографию.
