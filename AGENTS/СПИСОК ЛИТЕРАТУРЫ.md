# Библиография (правила проекта)

Этот файл фиксирует практические правила ведения библиографии в проекте.
Цель: чтобы `source/references.bib` компилировался `biber` без сюрпризов и выдавал
предсказуемый ГОСТ-подобный список источников.

## Текущая реализация в проекте

- Движок: `xelatex` (допускается `lualatex`).
- Библиография: `biblatex` + `biber`.
- Стиль: `gost-numeric`, сортировка отключена (`sorting=none`).
- Заголовок списка источников задаётся в `source/preamble.tex` и печатается через `\printbibliography`.
- Все записи лежат в `source/references.bib` (сейчас 20 записей).

## Каноничные требования к качеству записей

- Авторы должны быть в поле `author`, а не “прятаться” в `title`.
- У `@online`/`@report` обязательно проверять `year` (частая ошибка).
- Убирать хвосты вида `/ ...` из `title` (это обычно мусор из автогенераторов).
- Для веб-источников использовать официальный первоисточник (доки проекта/организации), а не пересказ.
- Wikipedia и аналогичные “живые” страницы для ссылок в работе не использовать.

## Поля, которые обычно нужны

Минимально полезные шаблоны (ориентир, не догма):

```bibtex
@book{key,
  author    = {Фамилия, И. О.},
  title     = {Название},
  year      = {2020},
  location  = {М.},
  publisher = {Издательство},
}

@online{key,
  title   = {Название страницы или документа},
  year    = {2024},
  url     = {https://example.org/...},
  urldate = {2026-04-23},
  note    = {Доп. сведения при необходимости},
}
```

## Как добавлять новый источник (быстро и безопасно)

1) Добавить запись в `source/references.bib` с понятным ключом (`snake_case`, без пробелов).
2) Проверить, что у записи есть `year`, а для онлайн — `url` и `urldate`.
3) Проставить `\cite{...}` в тексте и пересобрать через `.build/build.sh` (или минимум `xelatex → biber → xelatex`).

## Типовые ошибки, которые ломают вид списка

- Поле `title` содержит “мусор” (части URL, слэши, маркетинговые хвосты).
- У онлайн-источников отсутствует `year` или дата обращения (`urldate`).
- Имена авторов записаны непоследовательно (инициалы/порядок/латиница/кириллица).
- Дубли ключей (`@...{same_key,...}`) или несоответствие `\cite{}` и ключа в `.bib`.

## Оговорка о требованиях кафедры

Методички иногда требуют долю “свежей” литературы (например, 70% за последние 3 года).
Это требование не автоматизируется LaTeX’ом, поэтому проверяется вручную по `year`.
