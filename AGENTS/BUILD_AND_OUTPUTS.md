# Сборка и раскладка файлов

## Назначение

- Этот файл фиксирует рабочую схему сборки и расположение артефактов.
- Использовать его вместе с `AGENTS/AGENTS.md`.

## Структура

- `source/` - все редактируемые `.tex` и `references.bib`.
- `data/theory/` - теоретические материалы и конспекты.
- `results/` - готовые PDF-результаты.
- `.build/` - временная зона для промежуточных файлов, логов и скриптов.

## Что должно получаться после сборки

- `results/diploma.pdf` - итоговый полный файл работы.
- `results/chapters/*.pdf` - отдельные PDF по крупным частям.
- В корне проекта не должно оставаться новых PDF-артефактов.

## Как собирать

- Основную сборку выполнять из `source/` через `XeLaTeX` и `Biber`.
- Для проверки без генерации PDF использовать `xelatex -no-pdf`.
- Промежуточные результаты основной сборки держать в `.build/full/`.
- Промежуточные результаты побочных сборок держать в `.build/chapters/`.

## Локальная практика

- Если требуется пересобрать главы отдельно, использовать скрипт из `.build/chapters/`.
- Если требуется полная сборка, использовать скрипт из `.build/build.sh`.
- Перед копированием готовых файлов в `results/` очищать только устаревшие PDF-артефакты, а не исходники.

## Запрещённые вещи

- Нельзя возвращать `Taskfile.yml` как основную точку входа.
- Нельзя оставлять `main.pdf` или `Diploma.pdf` в корне проекта.
- Нельзя складывать промежуточные файлы в `results/`.
- Нельзя смешивать исходники и собранные PDF в одном каталоге.
