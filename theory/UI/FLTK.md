# FLTK.md

Этот файл описывает нативное desktop-приложение на FLTK на основе
`описание диплома.md`.
Документ нужен как опорный материал для раздела диплома о пользовательском
интерфейсе.

## Назначение

FLTK-версия представляет собой нативный настольный интерфейс для
инженерного использования вычислительного ядра.
Основной сценарий применения:

- быстрый интерактивный расчёт;
- табличная обработка наборов точек;
- построение диаграмм;
- экспорт результатов.

Интерфейс организован в виде вкладок (например, «Одиночный расчёт», «Табличный расчёт»,
«Графики», «О программе»), что соответствует типовым инженерным сценариям:
быстрый расчёт одной точки, пакетная обработка и визуальный анализ.

Приложение использует то же ядро `if97_core`, что и остальные каналы
доставки.
За счёт этого обеспечивается совпадение численных результатов.

## Основные функции

Поддерживаются:

- одиночный расчёт по выбранному режиму ввода;
- табличный расчёт с построчной обработкой ошибок;
- загрузка и вставка текстовых таблиц;
- реактивный парсинг табличных данных из буфера обмена;
- сохранение результатов как datasets;
- построение термодинамических диаграмм;
- отображение купола насыщения;
- экспорт логов, таблиц и графических материалов.

Это делает FLTK-приложение удобным для демонстрации практической ценности
ядра в дипломе.

## Архитектурная схема

Внутри FLTK-крейта можно выделить несколько зон ответственности.

`fltk/src/state.rs`

- хранит `AppState`;
- описывает сообщения и типы данных;
- управляет точками, таблицами и наборами данных.

Для повышения надёжности UI-часть реализована в духе MVU (Model--View--Update):
взаимодействие между обработчиками событий и состоянием приложения строится на
обмене сообщениями (через `fltk::app::channel`), без «расползания» общей
мутабельности по коду.

`fltk/src/ui/*`

- вкладки одиночного расчёта;
- вкладки табличного режима;
- вкладки диаграмм;
- служебные экраны и раздел about.

`fltk/src/plot/renderer.rs`

- подготовка точек для диаграмм;
- построение купола насыщения;
- экспорт графиков.

## Режимы работы

Одиночный режим нужен для интерактивного ввода одной точки состояния.
Пользователь выбирает входную пару и получает полный набор вычисленных
свойств.

Табличный режим нужен для пакетного расчёта нескольких строк.
В этом режиме особенно важна построчная локализация ошибок, потому что в
реальных инженерных данных одна некорректная строка не должна ломать весь
набор результатов.

Графический режим нужен для анализа состояния рабочей среды на диаграммах.
Он помогает не только получить численные значения, но и визуально
интерпретировать положение точки относительно области насыщения.

## Работа с диаграммами

Визуализация реализована с использованием `plotters`.
Для диплома важно подчеркнуть следующие особенности:

- отображаются пользовательские точки и наборы данных;
- поддерживается купол насыщения;
- есть автоподстройка масштаба;
- возможно сохранение графических артефактов.

Купол насыщения строится адаптивно, чтобы не перегружать рисунок лишними
точками и при этом не терять форму кривых.

Для устранения мерцания и артефактов отрисовка выполняется в буфер (например,
через `BitMapBackend`), после чего готовое изображение отображается в UI.
При сильном масштабировании применяются дополнительные меры устойчивости:
отсечение линий по алгоритму Коэна--Сазерленда и фильтрация `NaN`-значений до
передачи данных в рендерер.

## Работа с данными

Приложение поддерживает концепцию datasets.
Это удобно для инженерного сценария, когда пользователь сравнивает:

- несколько режимов работы;
- несколько наборов расчётных точек;
- разные серии экспериментов или расчётов.

Управление видимостью наборов данных на диаграммах позволяет использовать
FLTK-версию как прикладной инструмент анализа, а не только как тестовую
оболочку вокруг ядра.

## Почему выбран FLTK

Для диплома аргументация выбора может строиться так:

- нативный desktop-интерфейс;
- быстрый запуск;
- достаточный набор UI-компонентов;
- удобство интеграции с Rust;
- пригодность для инженерного рабочего места.
- запуск на различных версиях Windows.

FLTK-версия особенно полезна как наглядное подтверждение того, что ядро
можно использовать без браузера, web-runtime и сетевой инфраструктуры.

## Команда запуска

```bash
cargo run -p if97_calculator_fltk --release
```

## Что важно подчеркнуть в дипломе

- FLTK-клиент является самостоятельным настольным приложением;
- интерфейс рассчитан не только на одиночный ввод, но и на таблицы и
  диаграммы;
- вся предметная логика остаётся в `if97_core`;
- UI играет роль thin-layer адаптера и средства визуального анализа;
- наличие экспорта и datasets повышает практическую ценность системы.
