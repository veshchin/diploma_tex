\begin{frame}[t]{Актуальность и цель работы}
\small
Расчёт свойств воды и водяного пара -- базовая инженерная задача для теплоэнергетики и смежных областей.
\begin{itemize}
  \item нужен единый расчётный контур по IAPWS-IF97;
  \item нужны прямые и обратные маршруты для инженерных входных пар;
  \item нужны одинаковые результаты в приложении, настольной оболочке и сервисе.
\end{itemize}
\textbf{Цель:} разработать вычислительную модель и программный комплекс на основе IAPWS-IF97.
\end{frame}

\begin{frame}[t]{Область расчёта}
\small
В презентации рассматривается рабочая область, покрывающая все реализованные режимы расчёта:
\begin{itemize}
  \item регионы 1 и 2: $T \in [273{,}15;\,623{,}15]$~К, $p \le 100$~МПа;
  \item регион 3: $T \in [623{,}15;\,863{,}15]$~К, $p \in [22{,}064;\,100]$~МПа;
  \item регион 4: $T \in [273{,}16;\,647{,}096]$~К, $p \in [611{,}657;\,22{,}064\cdot 10^{6}]$~Па;
  \item регион 5: $T \in [1073{,}15;\,2273{,}15]$~К, $p \le 50$~МПа.
\end{itemize}
Точные границы важны не только для физической корректности, но и для маршрутизации расчёта и обработки ошибок.
\end{frame}

\begin{frame}[t]{Термодинамические потенциалы}
\small
В регионах 1, 2 и 5 свойства вычисляются через безразмерную энергию Гиббса:
\[
\gamma(\pi,\tau)=\frac{g(p,T)}{RT}, \qquad
\pi=\frac{p}{p^{*}}, \qquad
\tau=\frac{T^{*}}{T}.
\]
Через частные производные \(\gamma\) получаются основные свойства:
\[
v=\frac{RT}{p}\,\pi\gamma_{\pi}, \qquad
h=RT\,\tau\gamma_{\tau}, \qquad
s=R(\tau\gamma_{\tau}-\gamma).
\]
\[
u=RT(\tau\gamma_{\tau}-\pi\gamma_{\pi}), \qquad
c_p=-R\tau^2\gamma_{\tau\tau}.
\]
В регионе 3 используется безразмерная энергия Гельмгольца:
\[
\phi(\delta,\tau)=\frac{f(\rho,T)}{RT}, \qquad
\delta=\frac{\rho}{\rho^{*}}, \qquad
\tau=\frac{T^{*}}{T}.
\]
Для неё давление выражается через производную по плотности, а остальные свойства -- через производные по \(\delta\) и \(\tau\):
\[
p=\rho RT\,\delta\phi_{\delta}, \qquad
h=RT(\tau\phi_{\tau}+\delta\phi_{\delta}), \qquad
s=R(\tau\phi_{\tau}-\phi).
\]
Именно поэтому одно и то же ядро может считать давление, энтальпию, энтропию, теплоёмкость и скорость звука без дублирования формул.
\end{frame}

\begin{frame}[t]{Региональная модель и опорные точки}
\small
Сердце модели -- разбиение области на пять регионов с отдельными уравнениями и правилами перехода.
\begin{itemize}
  \item Регион 1: сжатая жидкость, уравнения Гиббса, вход \((p,T)\).
  \item Регион 2: перегретый пар, уравнения Гиббса, вход \((p,T)\).
  \item Регион 3: плотная область вблизи критической точки, уравнение Гельмгольца, вход \((\rho,T)\).
  \item Регион 4: линия насыщения, связь жидкой и паровой ветвей и расчёт смеси через \(x\).
  \item Регион 5: высокотемпературный пар, уравнения Гиббса.
\end{itemize}
\textbf{Тройная точка:} \(T_t=273{,}16\)~К, \(p_t=611{,}657\)~Па. Это нижняя опорная точка шкалы и начало линии насыщения.

\textbf{Критическая точка:} \(T_c=647{,}096\)~К, \(p_c=22{,}064\)~МПа, \(\rho_c=322\)~кг/м\textsuperscript{3}. Здесь исчезает различие между жидкостью и паром, а обратные задачи становятся плохо обусловленными.
\end{frame}

\begin{frame}[t]{Численные расширения модели}
\small
\begin{itemize}
  \item В однофазных областях 1, 2 и 5 маршрут \((\rho,T)\) реализован как численное расширение: сначала подбирается давление методом секущих, затем выполняется стандартный расчёт по \((p,T)\).
  \item В регионе 3 при задании \((p,T)\) сначала восстанавливается плотность из уравнения \(p_{\mathrm{if97}}(\rho,T)-p_{\mathrm{target}}=0\); для задач \((p,h)\) и \((p,s)\) используется двумерный решатель Левенберга--Марквардта.
  \item В регионе 4 при заданных \(\rho\) и \(T\) сначала находится \(p_{\mathrm{sat}}(T)\), затем степень сухости
  \[
  x=\frac{1/\rho-1/\rho'}{1/\rho''-1/\rho'}.
  \]
\end{itemize}
Так модель покрывает и аналитические, и численные маршруты без подмены физической основы стандарта.
\end{frame}

\begin{frame}[t]{Численные методы и сложность}
\small
\begin{itemize}
  \item Метод секущих даёт стоимость порядка \(O(k)\) на один запрос, а для таблицы из \(N\) строк -- \(O(Nk)\).
  \item Для околокритической области и обратных маршрутов \((p,h)\) и \((p,s)\) используется Левенберг--Марквардт с демпфированием и стартовым поиском.
  \item Аналитические обратные формулы используются там, где они заданы стандартом; численные методы включаются только в местах, где этого требует физика области.
\end{itemize}
\end{frame}

\begin{frame}[t]{Архитектура решения}
\small
Все вычисления сосредоточены в одном ядре, а пользовательские и сервисные слои выступают как тонкие адаптеры.
\begin{columns}[T,onlytextwidth]
  \column{0.32\textwidth}
  \textbf{if97\_core}\par
  \vspace{0.35em}
  Доменная модель, маршрутизация по регионам, прямые и обратные расчёты.

  \column{0.32\textwidth}
  \textbf{if97\_app\_api}\par
  \vspace{0.35em}
  Общий слой DTO и стабильный контракт обмена между компонентами.

  \column{0.32\textwidth}
  \textbf{Потребители}\par
  \vspace{0.35em}
  FLTK-клиент, Yew/WASM + Tauri, gRPC-сервис.
\end{columns}
\end{frame}

\begin{frame}[t]{Что реализовано в ядре}
\small
\begin{itemize}
  \item строго типизированные единицы измерения и безопасные новые типы;
  \item региональная модель IAPWS-IF97 с разбиением на 5 областей;
  \item расширение маршрутов по \((\rho,T)\) и итерационные решатели для обратных задач;
  \item статические таблицы коэффициентов, генерируемые на этапе сборки;
  \item явная обработка ошибок вместо неявных `NaN`-сценариев.
\end{itemize}
\end{frame}

\begin{frame}[t]{Что важно в интерфейсах}
\small
\begin{itemize}
  \item FLTK-версия закрывает быстрые автономные сценарии: одиночный и табличный расчёт, диаграммы и экспорт.
  \item Tauri-версия дает настольную оболочку на базе Yew/WASM с общим DTO и доступом к системным возможностям.
  \item Табличный режим обрабатывает строки поштучно и не останавливает весь расчёт из-за одной ошибки.
  \item Диаграммы поддерживают контроль режима и подготовку иллюстраций для отчётов.
\end{itemize}
\end{frame}

\begin{frame}[t]{Сервисный контур}
\small
\begin{itemize}
  \item сервисный контур реализован компонентом \texttt{if97\_calculator\_service} и использует `gRPC` для пакетной обработки;
  \item формат `Struct-of-Arrays` уменьшает накладные расходы на передачу данных;
  \item `tokio` отвечает за сетевой ввод-вывод, `rayon` - за вычислительный пул;
  \item `tonic-health` используется для статусов `Serving/NotServing` и корректного `drain`-завершения;
  \item ограничения очередей и батчей задаются переменными окружения.
\end{itemize}
\end{frame}
